# Customer Support Triage Environment

## Description
This OpenEnv project simulates a real-world Customer Support Triage system. An AI agent is tasked with evaluating incoming support tickets, determining the correct course of action, and appropriately routing or closing the tickets. It tests an agent's ability to comprehend text, prioritize critical issues (like security flaws), and apply appropriate schema restrictions.

## Action Space
The agent outputs a JSON matching the `Action` model:
- `ticket_id`: String ID of the ticket.
- `action_type`: "ROUTE" or "REPLY_AND_CLOSE"
- `department`: (Optional) "BILLING", "TECH_SUPPORT", "REFUNDS", "SECURITY"
- `reply_template`: (Optional) "REFUND_PROCESSED", "ESCALATED_TO_TECH", "GENERIC_APOLOGY"

## Observation Space
- `open_tickets`: Array of tickets currently requiring action.
- `last_action_feedback`: String indicating success or error of the previous action.
- `current_score`: Float representing cumulative reward.

## Tasks
1. **Easy**: Route a single, obvious technical support ticket.
2. **Medium**: Route two mixed tickets (billing and technical) accurately.
3. **Hard**: Manage three simultaneous tickets, identifying and prioritizing a hidden critical security vulnerability to the Security department while handling routine refunds and tech drops.
